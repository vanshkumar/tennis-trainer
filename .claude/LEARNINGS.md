# LEARNINGS

This file is the repo's running log of friction points hit by Claude (or other coding agents) and the signposts that resolved them. The goal is to make each session smarter than the last so the same mistakes don't recur.

## What to log here

- **Friction points**: build failures, test hangs, logic errors, surprising tooling behavior. Include the root cause and the specific fix.
- **Signposts after corrections**: any time the user corrects an approach or provides guidance you wouldn't have figured out independently, capture the rule and the reason. These let future sessions skip directly to the right answer.
- **Validated non-obvious choices**: when a slightly unusual approach turns out to be the right one and the user confirms it, log that too — otherwise future sessions may drift back toward the safer-looking but worse default.

## What not to log here

- Things that are already obvious from reading the code or git history.
- One-off task state (use the TaskCreate tool or the plan file for that).
- Anything covered in `CLAUDE.md` files or other repo docs.
